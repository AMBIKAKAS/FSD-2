Experiment No.: 8
Title: Develop RESTful APIs using Backend Framework (Flask)

Aim:

To design and develop RESTful APIs using the Flask backend framework for performing CRUD operations on student data.

----------------------------------------------------------------------------------------------

Objective

To understand the concept of REST architecture.

To implement backend services using the Flask framework.

To create API endpoints for Create, Read, Update, and Delete (CRUD) operations.

To learn the use of Blueprints for modular routing.

To test APIs using browser and curl/Postman.

📚 Tools and Technologies Used

Python 3

Flask

VS Code

Virtual Environment (venv)

Terminal / Command Line

-----------------------------------------------------------------------

Theory 

REST (Representational State Transfer) is an architectural style used to design networked applications. RESTful APIs use HTTP methods such as GET, POST, PUT, and DELETE to perform operations on resources. Flask is a lightweight Python web framework used to build web applications and APIs quickly and efficiently.

---------------------------------------------------------------------------------

Procedure

Created the project folder structure inside FSD-2/Experiment-8.

Created a virtual environment using python3 -m venv virenv.

Activated the virtual environment.

Installed Flask using pip.

Created the required files: app.py, run.py, and student_routes.py.

Implemented RESTful API endpoints for student management.

Registered Blueprint in the Flask application.

Started the backend server using python3 run.py.

Tested all API endpoints using browser and curl.

----------------------------------------------------------------------------------------------------

Learning Outcomes

After performing this experiment, I was able to:

Understand the working of RESTful APIs.

Develop backend services using Flask.

Implement CRUD operations through HTTP methods.

Use Flask Blueprints for modular code structure.

Create and manage virtual environments.

Test API endpoints using curl and browser.

Debug and run a Flask backend server successfully.
